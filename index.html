About Martin

화면을 만드는 개발자가 아니라, 운영 가능한 시스템의 구조를 설계하는 개발자입니다.

실무형 소프트웨어는 UI 하나가 아니라 업무 흐름, 데이터 구조, 시각화 인터페이스가 함께 맞물릴 때 완성된다고 생각합니다.

저는 웹 ERP, 데이터베이스, 그래픽 분석 프로그램을 개발하며 단순한 화면 구현보다 실제 업무 흐름과 데이터 구조를 이해하고 시스템화하는 데 강점이 있습니다.

ERP, 물류, 데이터베이스, 분석 도구, 디지털 트윈, AI Agent 자동화까지 여러 기술을 연결해 현장에서 바로 사용할 수 있는 소프트웨어를 만드는 것을 지향합니다.

MBTI는 ENTJ이며, 목표를 먼저 정의한 뒤 필요한 구조와 실행 계획을 빠르게 설계하고 구체적인 결과물로 전환하는 방식으로 일합니다.

Development Philosophy

개발 철학은 추상적인 슬로건보다 실제 설계 원칙처럼 읽히도록 정리했습니다.

1

복잡한 업무를 단순한 흐름으로 정리한다.

실무의 복잡성을 분석해 운영 가능한 절차와 화면 구조로 변환합니다.

2

흩어진 데이터를 연결 가능한 구조로 만든다.

저장만 되는 데이터가 아니라 검색과 분석, 자동화까지 이어질 수 있는 구조를 설계합니다.

3

시각화를 통해 시스템 상태를 직관적으로 이해하게 한다.

숫자와 로그를 현장에서 바로 읽을 수 있는 화면과 그래픽 도구로 바꾸는 데 집중합니다.
